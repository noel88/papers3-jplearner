# PaperS3 일본어 학습기 설계 문서

## 1. 프로젝트 개요

### 1.1 목적
M5Paper S3 기반 일본어 학습 전용 기기

### 1.2 대상 사용자
- 일본어 중상급 (N2-N1) 학습자
- 책상 상시 배치하여 수시로 학습

### 1.3 하드웨어
- **기기**: M5Paper S3 (ESP32-S3)
- **화면**: 960 x 540 E-ink (가로 모드 고정)
- **입력**: 터치스크린
- **저장**: 16MB 내장 플래시 + 16GB SD 카드
- **배터리**: 1800mAh 내장 (잔량 표시 지원)

## 4. 필사 기능

### 4.1 개요
- 365.epub 전용 (일일 명언/필사 콘텐츠)
- RTC 날짜 기반 오늘 챕터 자동 이동
- 전체 내용 스크롤 표시

### 4.2 오늘 날짜 자동 이동

```
RTC에서 날짜 읽음 (3월 18일)
        ↓
일수 계산 (31 + 28 + 18 = 77일)
        ↓
365.epub Chapter 077로 자동 이동
```

### 5.5 epub 기술 구조

```
epub 파일 = ZIP 압축
├── META-INF/
├── OEBPS/
│   ├── content.opf    # 메타데이터 (TinyXML2로 파싱)

└── mimetype
```

- **압축 해제**: miniz 라이브러리
- **XML 파싱**: TinyXML2
- **메모리**: PSRAM 활용

## 6. SRS 엔진

### 6.1 알고리즘
- **현재**: SM-2 (Anki 표준)
- **추후**: FSRS 업그레이드 고려

### 6.2 복습 버튼

| 버튼 | 의미 | 간격 영향 |
|------|------|----------|
| Again | 틀림/모름 | 처음부터 (1분 후) |
| Hard | 어려웠음 | 간격 × 1.2 |
| Good | 적당함 | 간격 × EF |
| Easy | 쉬웠음 | 간격 × EF × 1.3 |

### 6.3 신규 카드 초기 간격

| 응답 | 다음 복습 |
|------|----------|
| Again | 1분 |
| Hard | 6분 |
| Good | 10분 → 1일 |
| Easy | 4일 |

### 6.4 학습량 설정
- 하루 새 카드: 5~100개 (기본 20)
- 하루 복습 상한: 50~무제한 (기본 무제한)
- 학습 시간 목표: 10분~2시간 (기본 없음)

### 6.5 자동 표시
- 슬립 해제 시 복습 카드 자동 표시
- 단어 + 문형 랜덤 섞어서 표시

### 6.6 데이터 구조

```json
{
  "id": "word_001",
  "type": "word | sentence",
  "front": "勉強",
  "back": "べんきょう / 공부",
  "ef": 2.5,
  "interval": 1,
  "due": "2026-03-19",
  "reps": 0,
  "lapses": 0,
  "created": "2026-03-18",
  "source": "필사/365.epub",
  "example": "毎日勉強しています。"
}
```

## 8. 문형 학습 화면

### 8.1 카드 형식
- **앞면**: 문형만 (~ざるを得ない)
- **뒷면**: 문형 + 뜻 + 예문

### 8.2 데이터 소스
- 내장 문형: N2 (~200개) + N1 (~150개)
- 사용자 저장: 필사/읽기에서 추가

### 8.3 자동 인식
필사/읽기에서 문장 저장 시 내장 문형과 매칭

```
"行かざるを得ない" 저장
        ↓
"~ざるを得ない" 매칭됨!
        ↓
┌─────────────────────────┐
│ 문형 발견: ~ざるを得ない │
│                         │
│ [문형으로 저장] [문장으로 저장] │
└─────────────────────────┘
```

매칭 없을 때: 그냥 문장으로 저장

### 8.4 뒷면 레이아웃

```
┌──────────────────────────────────────────────────────────┐
│ 남은 카드: 15                                            │

│                  ~하지 않을 수 없다                       │
│                                                          │
│   例) 行かざるを得ない / 가지 않을 수 없다                 │
```

## 10. 설정 화면

### 10.1 구조
- 목록 형식 표시
- 클릭 시 상세 화면 이동
- 하단에 SD 카드 정보 고정 (읽기 전용)

### 10.2 설정 항목

| 카테고리 | 항목 | 기본값 | 옵션 |
|----------|------|--------|------|
| **언어** | UI 언어 | 한국어 | 한국어, 일본어 |
| **학습량** | 하루 새 카드 | 20개 | 5~100 |

| **단어/문형** | 카드 방향 | 한자→뜻 | 한자→뜻, 뜻→한자 |
| **단어/문형** | 예문 표시 | 항상 | 항상, 숨김, 터치시 |

## 11. 시스템 동작

### 11.1 부팅 시퀀스

```
전원 ON
    ↓
펌웨어 로드 (내장 플래시)
    ↓
SD 카드 마운트
    ↓
설정 로드 (내장 플래시)
    ↓
시작 화면 표시 (설정에 따라)
```

### 11.2 슬립 해제 시

```
슬립 해제
    ↓
복습할 카드 있음? ─No─→ 마지막 보던 화면
    │
   Yes
    ↓
단어/문형 복습 화면 자동 표시
(단어+문형 랜덤 섞어서)
```

### 11.3 텍스트 저장 플로우

```
필사/읽기 화면에서 텍스트 선택
    ↓
팝업 메뉴: [검색] [저장] [문형]
    ↓
[저장] 선택 → 단어장 추가 (SRS)
[문형] 선택 → 문형 매칭 확인
    ↓
매칭됨? → [문형으로 저장] / [문장으로 저장]
매칭 없음? → 문장으로 저장
```

## 14. 구현 우선순위 (제안)

### Phase 1: 기본 구조
1. SD 카드 마운트 및 파일 구조
2. 탭 바 네비게이션 (6탭 + 배터리)
3. 설정 화면 (기본)

### Phase 2: 필사/읽기 기능
1. epub 파서 통합 (miniz + TinyXML2)
2. 필사 화면 (365.epub, 오늘 날짜 자동 이동)
3. 읽기 화면 (책 선택 그리드)
4. 텍스트 선택 기능
5. 팝업 메뉴

### Phase 3: SRS 시스템
1. SM-2 알고리즘 구현
2. 단어 학습 화면
3. 문형 학습 화면
4. 슬립 해제 시 자동 표시

### Phase 4: 사전 연동
1. 오프라인 사전 로드
2. 단어 검색 기능
3. 문형 자동 인식

### Phase 5: 통계 및 마무리
1. 통계 화면
2. 진행률 저장
3. 배터리 잔량 표시
